```javascript
import fs from 'node:fs';
import { execSync } from 'node:child_process';

function getFdCount() {
  try {
    // Linux: -1 to exclude the readdir FD itself (pitfall 4 in research)
    return fs.readdirSync('/proc/self/fd').length - 1;
  } catch {
    // macOS / BSD
    const lines = execSync(`lsof -p ${process.pid} 2>/dev/null`).toString().trim().split('\n');
    return Math.max(0, lines.length - 1); // -1: subtract header row
  }
}
```

Create test/fd-leak.mjs as a Node.js ESM script. The test/ directory may not exist yet — create it.

The script must:

1. Import createRtpHandler using a relative ESM import: `import { createRtpHandler } from '../src/rtp/rtpHandler.js';`

2. Define a cross-platform getFdCount() function:
   - Try `fs.readdirSync('/proc/self/fd').length - 1` (Linux; subtract 1 for the readdir FD itself — see Pitfall 4 in research)
   - Catch and fall back to `execSync('lsof -p PID 2>/dev/null')` line count minus 1 (macOS header row)
   - Use delta comparison (baseline vs final) — this absorbs any ±1 off-by-one between calls

3. Define a no-op logger object so pino is not required at test time:
   ```javascript
   const noopLog = { info: () => {}, warn: () => {}, error: () => {}, debug: () => {}, trace: () => {} };
   ```

4. Main async function:
   a. Take ONE baseline FD reading (fdBefore)
   b. Loop 20 times: await createRtpHandler({ portMin: 20000, portMax: 20099, log: noopLog }), then immediately call rtp.dispose()
   c. Wait 50ms after the loop (allow GC/OS to reclaim async close completions): await new Promise(r => setTimeout(r, 50))
   d. Take final FD reading (fdAfter)
   e. Compute delta = fdAfter - fdBefore
   f. If delta > 2 (allow ±2 tolerance for Node internals): print error message with delta, process.exit(1)
   g. If delta <= 2: print success message with fdBefore, fdAfter, delta, process.exit(0)

5. Call main() and handle top-level errors with process.exit(1).

Use the port range 20000–20099 (distinct from production range 10000–10100) to avoid conflicts if the production service is running.

Tolerance of ±2: Node.js can hold 1-2 extra FDs transiently (DNS resolver, etc.). Using delta comparison and ±2 tolerance makes the test robust without masking real leaks (a leaked socket adds at minimum 1 FD per call = 20 leaked FDs after 20 calls, well above threshold).

The script should print human-readable output (not JSON), e.g.:
```
FD leak test: 20 sequential RTP allocations
  Baseline FDs: 41
  Final FDs:    41
  Delta:        0
PASS: no FD leak detected
```

or on failure:
```
FAIL: FD leak detected — delta=20 after 20 calls (expected <= 2)
```
