Make three targeted changes to session.go. Do NOT change any protocol behavior — existing audio forwarding and silence fallback must work identically after this commit.

**Step 1 — Add outboundFrame type** (add before the CallSession struct, after the pcmuSilenceFrame var):
```go
// outboundFrame is a tagged union for packetQueue entries.
// Exactly one field is set per instance:
//   - audio != nil: a 160-byte PCMU audio frame to send to the caller
//   - mark != "": a mark sentinel — route to markEchoQueue, send no RTP packet
type outboundFrame struct {
    audio []byte // non-nil for PCMU frames (160 bytes)
    mark  string // non-empty for mark sentinel frames
}
```

**Step 2 — Update CallSession struct**:
- Change `packetQueue chan []byte` to `packetQueue chan outboundFrame`
- Add after the dtmfQueue line:
  ```go
  markEchoQueue chan string     // mark names from rtpPacer to wsPacer (capacity 10)
  clearSignal   chan struct{}   // buffered (1): wsToRTP notifies rtpPacer to drain packetQueue
  ```

**Step 3 — Update run() initialization block** (after the s.dtmfQueue line):
- Change `s.packetQueue = make(chan []byte, packetQueueSize)` to `s.packetQueue = make(chan outboundFrame, packetQueueSize)`
- Add immediately after:
  ```go
  s.markEchoQueue = make(chan string, 10)
  s.clearSignal = make(chan struct{}, 1)
  ```

**Step 4 — Update wsToRTP packetQueue producer**:
In the chunking loop inside `case "media":`, change the select case from:
```go
case s.packetQueue <- chunk:
```
to:
```go
case s.packetQueue <- outboundFrame{audio: chunk}:
```
(There is only one enqueue site for audio in wsToRTP — the chunking loop.)

**Step 5 — Update rtpPacer packetQueue consumer**:
Replace the existing dequeue block inside `case <-ticker.C:`:
```go
// OLD:
var chunk []byte
select {
case chunk = <-s.packetQueue:
default:
    chunk = pcmuSilenceFrame
}
```
with:
```go
// NEW: dequeue next frame, fall back to silence if queue is empty
var frame outboundFrame
select {
case frame = <-s.packetQueue:
default:
    // empty — send silence below
}
chunk := frame.audio
if chunk == nil {
    chunk = pcmuSilenceFrame
}
```
The pkt construction, Marshal, and WriteTo calls below this block remain unchanged — they still use `chunk`.

IMPORTANT: Do not add mark sentinel routing or clearSignal handling in this plan — those come in Plan 02. After Step 5 the pacer will dequeue outboundFrame values but treat all of them as audio (frame.audio == nil falls back to silence). This is intentional and correct for Plan 01.
