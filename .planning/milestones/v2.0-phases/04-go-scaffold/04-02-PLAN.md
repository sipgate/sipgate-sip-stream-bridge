Replace the existing Node.js Dockerfile entirely with the Go two-stage build. The existing Dockerfile (Node.js 22 pnpm multi-stage) is fully superseded — git history is the backup.

Write `Dockerfile`:

```dockerfile
# sipgate-sip-stream-bridge Dockerfile — Go static binary (v2.0)
# Stage 1: Build — golang:1.26-alpine has musl libc + CA certs; produces statically-linked binary
# Stage 2: Runtime — FROM scratch (empty filesystem); only the binary and CA certs are included

# ── Stage 1: Build ───────────────────────────────────────────────────────────
FROM golang:1.26-alpine AS builder

WORKDIR /app

# Download dependencies first — this layer caches unless go.mod/go.sum changes
COPY go.mod go.sum ./
RUN go mod download

# Copy source and build static binary
COPY . .
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build \
    -ldflags="-s -w" \
    -trimpath \
    -o /sipgate-sip-stream-bridge \
    ./cmd/sipgate-sip-stream-bridge

# ── Stage 2: Runtime ─────────────────────────────────────────────────────────
# FROM scratch = empty filesystem; only our binary and CA certs are present.
# No shell, no libc, no package manager — minimal attack surface.
FROM scratch

# Copy CA certificates from the Alpine builder stage.
# REQUIRED for Phase 6: wss:// WebSocket TLS connections will fail without this.
# FROM scratch has no /etc/ssl/ directory — we must copy it explicitly.
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

# Copy the statically-linked binary
COPY --from=builder /sipgate-sip-stream-bridge /sipgate-sip-stream-bridge

# SIP ports — documented for reference; not required with network_mode: host
# Note: DO NOT EXPOSE the RTP port range (10000-10099/udp) here —
# large port ranges stall Docker Desktop's port proxy on macOS.
# RTP EXPOSE is added in Phase 6 with the macOS override pattern.
EXPOSE 5060/udp
EXPOSE 5060/tcp

ENTRYPOINT ["/sipgate-sip-stream-bridge"]
```

Build flags explained (for future readers):
- `CGO_ENABLED=0`: disables C bindings — produces a truly static binary (DCK-01)
- `GOOS=linux GOARCH=amd64`: explicit cross-compile target; safe on macOS M-series ARM hosts
- `-ldflags="-s -w"`: strips symbol table (-s) and DWARF debug info (-w); reduces binary 25–35%
- `-trimpath`: removes local filesystem paths from binary (reproducible builds, no path leakage)

Write the updated `docker-compose.yml`:

```yaml
# docker-compose.yml — Linux production configuration (v2.0 Go binary)
#
# network_mode: host gives the container direct access to the host network stack.
# This is REQUIRED for RTP media (Phase 6): Docker's port-proxy adds ~10ms jitter
# to every UDP packet and prevents the container from binding to ephemeral UDP ports.
#
# macOS / Windows: network_mode: host is silently ignored by Docker Desktop.
# For RTP testing on macOS, use docker-compose.override.yml:
#
#   services:
#     sipgate-sip-stream-bridge:
#       network_mode: bridge
#       ports:
#         - "5060:5060/udp"
#         - "5060:5060/tcp"
#         - "10000-10099:10000-10099/udp"
#
# WARNING: Restrict the UDP port range to ≤100 ports in the override — publishing
# large ranges causes Docker Desktop's port proxy to stall on startup.
#
services:
  sipgate-sip-stream-bridge:
    build: .
    image: sipgate-sip-stream-bridge:latest
    network_mode: host
    env_file:
      - .env
    restart: unless-stopped
```
