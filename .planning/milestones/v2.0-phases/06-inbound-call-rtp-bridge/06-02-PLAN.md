<objective>
RTP port pool (TDD) + CallManager + CallSession — the concurrent call session infrastructure.

Purpose: Provides the per-call resource management that enables CON-01 (independent sessions) and CON-02 (no FD/goroutine leaks). PortPool uses a buffered channel for O(1) acquire/release without locks. CallManager stores active sessions in a sync.Map. CallSession owns the UDPConn, WS connection, and two goroutines — all cleaned up via defers when the dialog context ends. session.go delegates WebSocket I/O to ws.go functions (which are created in 06-03 but stub-compiled here).

Output: internal/bridge/manager.go (PortPool + CallManager), internal/bridge/session.go (CallSession + run() lifecycle), internal/bridge/manager_test.go (TDD for PortPool).

    PortPool implementation:
    ```go
    type PortPool struct {
        ports chan int
    }

    func NewPortPool(min, max int) (*PortPool, error) {
        if min >= max {
            return nil, fmt.Errorf("RTP_PORT_MIN (%d) must be less than RTP_PORT_MAX (%d)", min, max)
        }
        ch := make(chan int, max-min)
        for p := min; p < max; p++ {
            ch <- p
        }
        return &PortPool{ports: ch}, nil
    }

    // Acquire is non-blocking — returns error if pool is exhausted (SIP-04).
    func (p *PortPool) Acquire() (int, error) {
        select {
        case port := <-p.ports:
            return port, nil
        default:
            return 0, fmt.Errorf("RTP port pool exhausted")
        }
    }

    // Release returns a port to the pool. Call via defer in StartSession.
    func (p *PortPool) Release(port int) {
        select {
        case p.ports <- port:
        default:
            // programming error: port was never acquired — drop silently
        }
    }
    ```

    ```go
    type CallSession struct {
        callID    string
        streamSid string
        dlg       *sipgo.DialogServerSession
        rtpPort   int
        callerRTP net.Addr      // *net.UDPAddr with caller's IP:port from SDP offer

        cfg       config.Config
        log       zerolog.Logger
        cancel    context.CancelFunc
        wg        sync.WaitGroup  // tracks rtpToWS + wsToRTP goroutines for CON-02
    }
    ```

    run(ctx context.Context) implementation — CRITICAL ordering around sendStop:
    1. Check dlg.Context().Err() != nil at entry — if already done, log and return (handles Pitfall 6: race where BYE arrives before session starts)
    2. sessionCtx, cancel := context.WithCancel(dlg.Context()) — session context derived from dialog
    3. s.cancel = cancel; defer cancel()
    4. rtpConn, err := net.ListenUDP("udp4", &net.UDPAddr{Port: s.rtpPort}) — if err: log + s.dlg.Bye(context.Background()) + return
    5. defer rtpConn.Close() — FD cleanup (CON-02)
    6. wsConn, err := dialWS(sessionCtx, s.cfg.WSTargetURL) — if err: log + s.dlg.Bye(context.Background()) + return (WSR-01 initial check)
    7. defer wsConn.Close()
    8. sendConnected(wsConn) — if err: log + return (session cleanup via defers)
    9. sendStart(wsConn, s.streamSid, s.callID, s.dlg.InviteRequest) — if err: log + return (note: dlg.InviteRequest is the *siplib.Request field on DialogServerSession)
    10. s.wg.Add(2); go s.rtpToWS(sessionCtx, rtpConn, wsConn); go s.wsToRTP(sessionCtx, wsConn, rtpConn)
    11. <-sessionCtx.Done() — blocks until BYE or WS stop
    12. s.wg.Wait() — MUST come BEFORE sendStop: drains rtpToWS goroutine first (rtpToWS is the sole WS writer — calling sendStop concurrently with rtpToWS would race on wsConn; gobwas/ws is not concurrent-write-safe per RESEARCH.md Pitfall 5)
    13. sendStop(wsConn, s.streamSid, s.callID) — safe to write now: rtpToWS is guaranteed done; log if error (best effort)

    WRITE-SAFETY INVARIANT: rtpToWS is the ONLY goroutine that writes to wsConn during the session. sendStop runs only after wg.Wait() confirms rtpToWS has exited. This makes run() the exclusive writer for sendStop and maintains the single-writer-at-a-time guarantee throughout the session's full lifetime.

    rtpToWS goroutine — follows RESEARCH.md Pattern 5:
    - defer s.wg.Done()
    - buf := make([]byte, 1500) — MTU-safe buffer
    - seqNo starts at 2 (connected=implicit, start=1)
    - startTimeMs := time.Now().UnixMilli()
    - Loop: SetReadDeadline(100ms) → ReadFromUDP → Unmarshal pion/rtp Packet → skip if PT == s.dtmfPT (DTMF, handled Phase 7) → skip if PT != 0 (non-PCMU) → base64 encode payload → writeJSON MediaEvent to wsConn
    - On ctx.Err() != nil: return (clean exit)
    - On timeout (net.Error.Timeout()): continue (re-check ctx then re-read)
    - On other read error: log + return

    wsToRTP goroutine — follows RESEARCH.md Pattern 6:
    - defer s.wg.Done()
    - ssrc := rand.Uint32(); var seqNo uint16; var timestamp uint32
    - Loop: check ctx.Err() → wsutil.ReadServerData(wsConn) → json.Unmarshal to map[string]json.RawMessage → extract "event" field → switch:
      - "media": decode base64 payload → build pion/rtp Packet (PT=0, seqNo, timestamp, SSRC=ssrc) → Marshal → WriteTo(rtpConn, s.callerRTP) → seqNo++; timestamp+=160
      - "stop": log + s.dlg.Bye(context.Background()) + s.cancel() + return (SIP-05)
    - On WS read error: log + s.dlg.Bye(context.Background()) + s.cancel() + return

    CRITICAL: Only wsToRTP reads from WS. Only rtpToWS (and run() after wg.Wait()) writes to WS.
    This prevents the concurrent write race identified in RESEARCH.md Pitfall 5.
