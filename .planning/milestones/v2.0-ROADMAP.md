# Milestone v2.0: Go Rewrite

**Status:** ✅ SHIPPED 2026-03-05
**Phases:** 4–8
**Total Plans:** 9

## Overview

Complete rewrite of audio-dock in Go — same external interface as v1.0, deterministic audio without GC jitter, ~1 MB Docker image from scratch, and Prometheus observability built in. All 29 v1 requirements satisfied. Drop-in replacement for the Node.js v1.0 implementation.

## Phases

### Phase 4: Go Scaffold

**Goal**: A runnable Go binary that validates all required environment variables at startup, logs structured JSON with zerolog, exits with a descriptive error on misconfiguration, and ships as a static FROM-scratch Docker image
**Depends on**: Nothing (first v2.0 phase)
**Requirements**: CFG-01, CFG-02, CFG-03, CFG-04, CFG-05, OBS-01, DCK-01, DCK-02, DCK-03
**Plans**: 2 plans

Plans:
- [x] 04-01-PLAN.md — Go module init + config validation (go-simpler/env) + zerolog entry point (CFG-01..05, OBS-01)
- [x] 04-02-PLAN.md — Two-stage Dockerfile (golang:1.26-alpine + FROM scratch) + docker-compose.yml update (DCK-01..03)

**Success Criteria:**
1. Binary without required env vars prints descriptive error and exits non-zero
2. Binary with all env vars starts and emits structured JSON log lines (zerolog format) to stdout
3. `docker build` produces static binary image using `FROM scratch`; image under 20 MB
4. `docker compose up` starts the service using Compose file with `network_mode: host`

**Goal**: Accept inbound SIP INVITE, negotiate PCMU, open per-call RTP socket, connect dedicated WebSocket, forward audio bidirectionally — full Twilio Media Streams protocol end-to-end; multiple simultaneous calls; all resources cleaned up on call end
**Depends on**: Phase 5
**Requirements**: SIP-03, SIP-04, SIP-05, WSB-01, WSB-02, WSB-03, WSB-04, WSB-05, WSB-06, WSR-01 (partial), CON-01, CON-02
**Plans**: 3 plans

Plans:
- [x] 06-01-PLAN.md — TDD SDP parsing (CallerSDP/ParseCallerSDP/BuildSDPAnswer) + SIP INVITE/ACK/BYE/OPTIONS handler with DialogServerCache (SIP-03, SIP-04, SIP-05)
- [x] 06-02-PLAN.md — TDD PortPool (channel-based bounded pool) + CallManager (sync.Map registry) + CallSession (per-call RTP socket + goroutine lifecycle + ws.go stubs) (CON-01, CON-02, WSR-01)
- [x] 06-03-PLAN.md — Full WebSocket bridge (Twilio connected/start/media/stop events via gobwas/ws) + main.go Phase 6 wiring (WSB-01..06)

**Success Criteria:**
1. Inbound call triggers `connected` then `start` event with correct streamSid, callSid, tracks, mediaFormat, and From/To/Call-ID in customParameters
2. Voice audio spoken appears as `media` events (base64 PCMU) on WS; audio sent as `media` is heard by caller
3. Caller hang-up → SIP BYE + `stop` event; WS `stop` → SIP BYE to caller
4. If WebSocket unreachable at call start → SIP BYE to caller; no RTP socket or goroutine leaked
5. Two simultaneous calls: independent WS + independent RTP socket; no cross-contamination; clean teardown

## Milestone Summary

**Key Decisions:**

- SIP library: emiago/sipgo v1.2.0 — pure Go, stable v1.x, production-proven via LiveKit
- SDP/RTP: pion/sdp v3.0.18 + pion/rtp v1.10.1 — same libraries LiveKit uses
- Logging: rs/zerolog — structured JSON, low-allocation hot path
- No diago: raw RTP byte access needed for Twilio base64 encoding; diago abstraction fights this
- DTMF PT: extracted from SDP offer dynamically (sipgate uses PT 113, not conventional 101)
- Docker final image: FROM scratch with CGO_ENABLED=0 static binary
- CallManagerIface defined in sip package to avoid circular import with bridge package
- wsSignal uses sync.Once to guard channel close — prevents double-close panic on simultaneous goroutine failures
- Custom prometheus.Registry — excludes Go runtime metrics; only 5 audio-dock metrics exposed on /metrics
- active_calls_total as Gauge with _total suffix — OBS-03 specifies name literally; Prometheus client allows it

**Issues Resolved:**

- sipgo graceful shutdown (#116) not built-in — RESOLVED with DrainAll BYE polling loop (LCY-01)
- sipgo.WithUserAgentName incorrect API — fixed to sipgo.WithUserAgent in 06-03 (live testing bug)
- SDP_CONTACT_IP made optional with auto-detect of local outbound IP (runtime fix)
- 4 runtime SIP/bridge bugs fixed during Phase 6 live testing (Record-Route stripping, 200 OK timing, CANCEL handling, User-Agent header)

**Issues Deferred:**

- Human verification items require live sipgate credentials (E2E audio, concurrent calls, registration confirm)
- DTMF PT mismatch: verify sipgate server-granted Expires on first live registration

**Technical Debt Incurred:**

- WSB-01..06 checkboxes not updated in REQUIREMENTS.md during Phase 6 (documentation gap only)
- Velocity tracking entry malformed in STATE.md (cosmetic)
