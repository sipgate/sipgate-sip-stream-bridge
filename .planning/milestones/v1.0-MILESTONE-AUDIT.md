## Tech Debt Summary

**Total items:** 5 (0 blockers)

### M-01: LOG_LEVEL Config Bypass (minor)
- **File:** `src/logger/index.ts:4`
- **Issue:** `process.env.LOG_LEVEL ?? 'info'` read directly instead of via validated `config.LOG_LEVEL`
- **Impact:** Zod-validated config value for `LOG_LEVEL` silently discarded; pino sees raw env string. Functionally identical since Zod does not transform the enum, but violates CFG-05 single-source-of-truth principle.
- **Affected requirements:** OBS-01, CFG-05
- **Fix:** Pass `config.LOG_LEVEL` into `createChildLogger` or export a `setLogLevel` helper.

### M-02: DTMF wsReconnecting Gate Absent (cosmetic)
- **File:** `src/bridge/callManager.ts:461`
- **Issue:** `rtp.on('dtmf')` handler has no `if (session.wsReconnecting) return` guard, unlike the audio handler at line 453
- **Impact:** None — `wsClient.ts:116` drops silently when `readyState !== OPEN`. Intent-vs-implementation gap makes the code harder to reason about.
- **Affected requirements:** WSB-07
- **Fix:** Add `if (session.wsReconnecting) return;` as first line of the dtmf handler.

### ROADMAP Tracking Inconsistency (stale data)
- **File:** `.planning/ROADMAP.md`
- **Issue:** Phase 1 top-level checkbox (`- [ ] **Phase 1: Foundation**`) is unchecked; plans 01-01 and 01-03 have unchecked boxes; Progress table shows "2/3 plans, In Progress". All three SUMMARY.md files exist and VERIFICATION.md shows 11/11 checks passing.
- **Impact:** Cosmetic only — GSD tools may misreport Phase 1 status.
- **Fix:** Update ROADMAP.md plan checkboxes and Progress table to reflect actual state.
