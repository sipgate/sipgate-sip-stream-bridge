## Milestone Summary

**Key Decisions:**
- outboundFrame tagged union: separate audio/mark fields over magic-byte sentinel — idiomatic nil check for silence fallback
- wsPacer sole-writer invariant: mark echoes route via markEchoQueue channel to wsPacer; wsToRTP/rtpPacer never write directly to wsConn
- RTP pacer never stopped on clear: drain packetQueue only; seqNo/timestamp advance on sentinel frames to prevent RTP gaps
- OPTIONS 401/407 not a failure: treat as "server alive, auth issue" — only timeout/5xx/404 trigger re-registration
- doRegister mutex mandatory: sipgo concurrent client.Do() thread-safety is undocumented; mutex serializes both reregisterLoop and optionsKeepaliveLoop
- CSeq header routing (Node.js): dispatches OPTIONS responses before REGISTER state machine — prevents OPTIONS 401/407 from triggering REGISTER auth challenge

**Issues Resolved:**
- Silent registration loss undetectable in v2.0 — resolved with OPTIONS keepalive in both Go and Node.js
- Barge-in protocol incomplete in v2.0 — resolved with full mark/clear implementation

**Tech Debt Incurred:**
- VALIDATION.md frontmatter not updated from draft (nyquist_compliant: false) — tests passed, cosmetic only
- OPTIONS timeout asymmetry: Go=10s, Node.js=5s — minor, both correct
- Human verification items: live sipgate OPTIONS response classification; SIGTERM goroutine-leak test
