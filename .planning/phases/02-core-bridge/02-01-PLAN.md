RTP header byte layout (RFC 3550 §5.1):
  Byte 0:  V(2) P(1) X(1) CC(4)
  Byte 1:  M(1) PT(7)
  Bytes 2-3:  Sequence Number (uint16 big-endian)
  Bytes 4-7:  Timestamp (uint32 big-endian)
  Bytes 8-11: SSRC (uint32 big-endian)
  Bytes 12+:  Payload

RFC 4733 telephone-event payload (4 bytes):
  Byte 0: Event code (0-9=digits, 10=*, 11=#, 12-15=A-D)
  Byte 1: bit7=End, bit6=reserved, bits5-0=Volume
  Bytes 2-3: Duration (uint16 big-endian)

Create `src/rtp/rtpHandler.ts`. Imports: `node:dgram`, `node:crypto`, `node:events`, pino Logger.

**Export `RtpHandler` interface:**
```typescript
export interface RtpHandler extends EventEmitter {
  localPort: number;
  /** Call after WS connected + start event sent — enables payload forwarding */
  startForwarding(): void;
  /** Send mulaw payload Buffer to the remote RTP endpoint */
  sendAudio(payload: Buffer): void;
  /** Release dgram socket, remove listeners */
  dispose(): void;
}
```

**Export `createRtpHandler(opts: { portMin: number; portMax: number; log: Logger }): Promise<RtpHandler>`:**

Port allocation (RESEARCH.md open question 2 recommendation):
- Use a module-level `let nextPort = portMin` counter (module singleton — safe because synchronous)
- Try to `socket.bind(nextPort)` — increment and retry on EADDRINUSE
- Wrap in EADDRINUSE catch: try next port up to (portMax - portMin) attempts; throw if all exhausted
- Log warn if fewer than 10 ports remain free: `Math.floor((portMax - nextPort) / 2) < 10`
- Resolve the Promise once `socket.bind()` succeeds

Per-call socket behavior:
- `dgram.createSocket('udp4')`
- Maintain private state: `forwardingEnabled = false`, `outSeq = 0`, `outTimestamp = 0`, `outSsrc = crypto.randomBytes(4).readUInt32BE(0)`
- `startForwarding()` sets `forwardingEnabled = true`
- On `socket.on('message', (buf) => {...})`:
  - Parse RTP header: `const byte0 = buf[0]; const csrcCount = byte0 & 0x0f; const extBit = (byte0 & 0x10) !== 0; const headerLen = 12 + csrcCount * 4 + (extBit ? 4 + buf.readUInt16BE(14) * 4 : 0)`
  - Read payloadType: `buf[1] & 0x7f`
  - If payloadType === 0 (PCMU) and `forwardingEnabled`: emit `'audio'` with `buf.subarray(headerLen)` (zero-copy)
  - If payloadType === 101 (telephone-event) and `forwardingEnabled`:
    - Payload: `buf.subarray(headerLen)` (must be ≥4 bytes)
    - `const isEnd = (payload[1] & 0x80) !== 0`
    - If isEnd: look up digit in map `{0:'0',...,9:'9',10:'*',11:'#',12:'A',13:'B',14:'C',15:'D'}`, emit `'dtmf'` with `{ digit }`
  - All other payload types: ignore silently (log.trace only, per Claude's discretion on error verbosity)

`sendAudio(payload: Buffer)`:
- Build 12-byte RTP header (RESEARCH.md Pattern 5):
  ```
  header[0] = 0x80  // V=2, P=0, X=0, CC=0
  header[1] = 0x00  // M=0, PT=0 (PCMU)
  header.writeUInt16BE(outSeq++, 2)
  header.writeUInt32BE(outTimestamp, 4)
  outTimestamp += 160  // 20ms at 8kHz
  header.writeUInt32BE(outSsrc, 8)
  ```
- Set remote address/port from `setRemote(ip, port)` call (see below)
- `socket.send(Buffer.concat([header, payload]), remotePort, remoteIp)`

`setRemote(ip: string, port: number)` — called by CallManager after SDP parsing; stored as `remoteIp`/`remotePort`

`dispose()`:
- Set `forwardingEnabled = false`
- `socket.removeAllListeners()`
- `socket.close()` (swallow ENOTCONN if already closed)

Implement `RtpHandler` by extending `EventEmitter`. Export type for events:
```typescript
// emits: 'audio' (payload: Buffer), 'dtmf' (data: { digit: string })
```

Anti-pattern note: Do NOT share the RTP socket across calls — each `createRtpHandler` call creates a fresh socket (RESEARCH.md anti-patterns).
