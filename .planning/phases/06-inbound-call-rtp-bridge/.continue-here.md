<context>
The core architecture is sound. The issue was purely the CANCEL race condition:
1. Caller sends CANCEL between our 180 Ringing and 200 OK
2. sipgo auto-handles it internally (200 to CANCEL, 487 to INVITE, dialog context canceled)
3. Our StartSession goroutine needs to detect this via dlg.Context().Err() after WS dial

With the guard in place, the flow is:
- 180 Ringing → WS dial (context.Background() timeout) → check dlg.Context().Err()
- If CANCEL: close WS, return (port released by defer)
- If OK: RespondSDP 200 OK → session.run(dlg.Context(), wsConn) → audio bridge
