# v3.0.0 — Mid-Call REST Control + B2BUA `<Dial>`

Released: 2026-05-XX

v3.0 layers a 100%-Twilio-compatible HTTP REST API for **modifying
calls already in progress** on top of v2.1's shipped SIP↔WebSocket
media bridge. Default behaviour is preserved: every inbound call
auto-answers and streams to `WS_TARGET_URL` (zero-config v2.1 parity).

This file is the operator-facing pitch; the canonical developer audit
trail is [`CHANGELOG.md`](../../CHANGELOG.md). When the two diverge,
the CHANGELOG wins.

## What's new at a glance

- **REST API for live call modification** — `POST /Calls/{CallSid}.json`
  with `Twiml=`, `Url=`, or `Status=completed`.
- **TwiML `<Dial>` for call forwarding** — B2BUA via the sipgate trunk;
  full Twilio attribute parity for `<Dial>` and `<Number>`.
- **Status callbacks with X-Twilio-Signature parity** — HMAC-SHA1
  signed; golden-vectored against twilio-python and twilio-node.
- **Operator-trusted default StatusCallback** —
  `STATUS_CALLBACK_DEFAULT_URL` bypasses the SSRF guard for fixed
  operator-side endpoints, while runtime-supplied URLs remain guarded.
- **Toll-fraud guardrails** — `DIAL_ALLOWED_PREFIXES` allow-list with
  default-deny posture; enforced at the SIP layer.
- **Privacy gate** — WS stream closes with `stop reason="dial-forward"`
  BEFORE the outbound INVITE on `<Dial>`; verified end-to-end by the
  e2e suite.
- **Observability discipline** — bounded label cardinality enforced by
  CI (`cmd/lint-metrics`); Grafana dashboard + Prometheus alert rules
  shipped under `deploy/observability/`.
- **Hardening** — single drain entry-point with dual-leg discipline,
  15 s drain budget, security-header middleware, 64 KB REST body cap,
  6.0 MB image-size CI gate (see `docs/operator/IMAGE_SIZE.md`),
  forwarder ring-timeout watchdog, operator runbook published.

## Migration from v2.1

**Zero config required for v2.1 deployments** — every existing `.env`
continues to work unchanged. The new env vars are all optional unless
you want to enable the corresponding feature.

### To enable the REST API

```env
AUTH_TOKEN=<your_twilio_compatible_secret>
PUBLIC_BASE_URL=https://your-bridge.example.com
```

`AUTH_TOKEN` is the Twilio-compatible Basic Auth secret (REST request
authentication) AND the HMAC-SHA1 signing key for outbound
`X-Twilio-Signature` headers. `PUBLIC_BASE_URL` is the externally-
reachable URL of the bridge — used for signature URL reconstruction
when the bridge runs behind a reverse proxy.

### To enable `<Dial>` toll-fraud guardrails

Mandatory if `<Dial>` is reachable from your bot's TwiML — without
this opt-in, every `<Dial>` is rejected (default-deny):

```env
DIAL_ALLOWED_PREFIXES=+49,+1,+44      # CSV of allowed E.164 prefixes
DIAL_DEFAULT_CALLER_ID=+4912345       # fallback caller-ID when TwiML callerId is unset
DIAL_CALLER_ID_COUNTRY_CODE=49        # E.164 country code (without +) for trunk display normalisation
```

Per-market recipes are documented in
[`docs/operator/TOLL_FRAUD.md`](../operator/TOLL_FRAUD.md) (Germany,
USA, UK, DACH, Western Europe, deny-all).

### Optional `<Dial>` rate limits

```env
DIAL_RING_TIMEOUT_S=30      # default ring timeout (5–600)
DIAL_MAX_PER_SESSION=3      # max <Dial> calls per inbound session
DIAL_MAX_PER_MINUTE=60      # global rolling-minute outbound dial cap
```

## New env vars

| Var                            | Default       | Description                                                                                       |
| ------------------------------ | ------------- | ------------------------------------------------------------------------------------------------- |
| `AUTH_TOKEN`                   | (unset)       | Twilio-compat Basic Auth + HMAC-SHA1 signer key                                                   |
| `PUBLIC_BASE_URL`              | (unset)       | Externally-reachable URL; used for X-Twilio-Signature URL reconstruction                          |
| `DIAL_ALLOWED_PREFIXES`        | `""` (deny-all) | CSV of allowed E.164 prefixes for `<Dial>` (default empty rejects every outbound dial)         |
| `DIAL_DEFAULT_CALLER_ID`       | (unset)       | Fallback caller-ID when TwiML `callerId` is unset and inbound-ANI cannot be preserved             |
| `DIAL_CALLER_ID_COUNTRY_CODE`  | (unset)       | E.164 country code (without `+`) used to normalise display caller-IDs to sipgate trunk format     |
| `DIAL_RING_TIMEOUT_S`          | `30`          | Default ring timeout in seconds (5–600); overridable per `<Dial>` via TwiML `timeout=` attribute  |
| `DIAL_MAX_PER_SESSION`         | `3`           | Max `<Dial>` calls per inbound session (rate-limit; per-call lifetime)                            |
| `DIAL_MAX_PER_MINUTE`          | `60`          | Global rolling-minute outbound dial cap (toll-fraud / overload defence)                           |

For the full credential rotation procedure (when to rotate, how to
verify, audit-log queries), see
[`docs/operator/INCIDENT_RESPONSE.md`](../operator/INCIDENT_RESPONSE.md).

- `github.com/emiago/sipgo` `1.2.0` → `1.3.0` (auto-401/407 in
  `WaitAnswer`).

## Removed / breaking changes since v2.1

- **Pre-answer webhook fetch path** — removed during the 2026-04-29
  re-scope.
- **`VOICE_URL`, `VOICE_METHOD`, `VOICE_FALLBACK_URL`** env vars —
  removed.
- **`mode` field in `/health` JSON** — `/health` is now LOCKED to the
  4-field shape: `{"registered", "account_sid", "active_calls",
  "active_forwards"}` (snake_case). Operators relying on `mode` MUST
  update their probes.

If your v2.1 deployment uses any of the above, see
[`docs/operator/RELEASE_CHECKLIST.md`](../operator/RELEASE_CHECKLIST.md)
for the migration smoke-verification checklist.

## Image-size note

Runtime Docker image (multi-stage build, `FROM scratch` runtime)
measures 5,466,112 bytes on v3.0 (~5.2 MB). The CI gate budget is
6,000,000 bytes — ~10 % margin against the measured baseline while
staying tight enough to catch material regressions (a profiler
addition or a new direct dependency >100 KB would still flip the
gate). A static sipgo + gobwas/ws + pion/{rtp,srtp,sdp} +
prometheus/client_golang + zerolog Go OCI layer cannot plausibly fit
under 1 MB even with `-ldflags="-s -w" -trimpath`.

Drivers of the v3.0 footprint vs. v2.1:

- Toolchain bumps (Go 1.24 → 1.26; sipgo 1.2 → 1.3).
- chi v5.2.5 router for the REST API.
- `golang.org/x/tools` for the cardinality lint binary.

The 6.0 MB budget is enforced as a build-blocking CI gate via
`make image-size-check`. Any future regression triggers a re-budgeting
review per
[`docs/operator/IMAGE_SIZE.md`](../operator/IMAGE_SIZE.md).
